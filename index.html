第一篇博客

发表于 2017-10-13

为了搭建这个感觉高大上的博客废了好多事，那第一篇博客就说说这个过程中遇到的坑吧（整个配置流程网上太多了就不说了）

1.傻逼的配置文件惹的祸
问题现象：使用hexo server本地访问可以正常访问，但是服务器上的就是访问不了，而且在命令的执行过程中没有报出任何的错误
问题原因：配置文件格式不正确
config.yml配置文件里的格式是在“:”后面都要有一个空格，然后再输入value，类似：
type: git
我一开始配置没有留任何空格，在生成和发布过程中也没有抛出异常（我们是程序员哎，只求报错，不求通过啊），然后就悲剧了
其实反过头想想，本地OK，服务器失败，肯定与发布有关，在配置文件中与发布相关的配置只有在hexo的config.yml文件中的deploy字段，再锁定deploy字段检查应该很快就可以锁定问题
